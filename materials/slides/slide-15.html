구조(Scaffolding)

역할별 폴더링 — 목적이 명확한 구조

my-project/
├── src/ # 비즈니스 로직
├── docs/ # AI의 참고 문서
├── tests/ # 검증 인프라
├── .dev/ # 개발 도구/스크립트
├── .claude/ # AI 설정 (skills, hooks, rules)
├── out/ # 빌드 산출물
└── CLAUDE.md # 프로젝트 지도

왜 폴더링이 중요한가

AI가 "테스트 작성해줘"라고 하면 tests/에, "문서 업데이트"라고 하면 docs/에 자동으로 찾아간다

docs/ = AI의 참고서

아키텍처 문서, 컨벤션, 스펙을 docs/에 놓으면 AI가 작업 전에 참고한다. 정적 문서가 아닌 살아있는 인프라.

out/ 분리

빌드 산출물을 분리해야 AI가 소스와 산출물을 혼동하지 않는다

폴더 이름만 명확해도 AI가 알아서 올바른 위치에 파일을 생성한다
